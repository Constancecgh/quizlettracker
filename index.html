📊 3. Représenter l'information

Tout ce qu'un ordinateur manipule est converti en nombres binaires. La différence c'est comment on interprète ces nombres.

Nombres
Des bits représentent directement des valeurs entières ou décimales. Ex : 01000001 = 65
Texte (ASCII)
Chaque nombre correspond à un caractère. 65 = 'A', 66 = 'B', 97 = 'a'… (table ASCII)
Couleurs (RGB)
3 octets = Rouge, Vert, Bleu. Chacun de 0 à 255. Ex : (255, 0, 0) = rouge pur
Images
Une grille de pixels, chaque pixel = une couleur RGB. 1920×1080 pixels = plus de 2 millions de valeurs RGB
Son / Vidéo
Des milliers d'échantillons par seconde, chacun encodé en binaire. Une vidéo = images + son synchronisés
💡 Le même octet 01000001 peut signifier le nombre 65, la lettre 'A', ou une composante de couleur. C'est le contexte qui donne le sens.
⚙️ 4. Algorithmes — résoudre des problèmes

Un algorithme est une suite d'instructions précises pour résoudre un problème. L'exemple classique de CS50 : trouver un nom dans un annuaire téléphonique.

Algorithme 1 — lent

Feuilleter page par page depuis le début.

1
Ouvrir à la page 1
2
Regarder le nom
3
Si trouvé → stop
4
Sinon → page suivante, retour à 2
→ n pages = n étapes
Algorithme 3 — rapide

Ouvrir au milieu, éliminer la moitié.

1
Ouvrir au milieu
2
Si nom trouvé → stop
3
Si nom avant → jeter moitié droite
4
Si nom après → jeter moitié gauche, retour à 1
→ n pages = log₂(n) étapes
📌 C'est exactement la recherche binaire de la Week 3. CS50 introduit ce concept dès la Week 0 avec l'annuaire pour que tu comprennes l'intuition avant le code.
🎭 5. Abstraction

L'abstraction consiste à cacher la complexité en dessous pour ne montrer que ce qui est utile. C'est le principe fondamental de toute l'informatique.

Application
Ce que tu utilises — "Envoyer un email", "Jouer de la musique", "Chercher sur Google"
Langage haut niveau
Python, JavaScript — proche du langage humain, cache les détails
Langage bas niveau
C, assembleur — plus proche de la machine, contrôle plus fin
Binaire / CPU
Ce que la machine comprend — 0 et 1, transistors, électricité
💡 Quand tu appelles printf() en C, tu n'as pas besoin de savoir comment ça affiche des pixels à l'écran. C'est l'abstraction en action : tu utilises une fonction sans en connaître les détails internes.
🔵 6. Logique booléenne

En informatique, tout repose sur des valeurs vrai (1) ou faux (0). Les opérateurs booléens combinent ces valeurs.

AND
Les deux doivent être vrais
1 AND 1 = 1
1 AND 0 = 0
0 AND 1 = 0
0 AND 0 = 0
OR
Au moins un doit être vrai
1 OR 1 = 1
1 OR 0 = 1
0 OR 1 = 1
0 OR 0 = 0
NOT
Inverse la valeur
NOT 1 = 0
NOT 0 = 1
📌 En C et Python tu les écriras && / || / ! (C) ou and / or / not (Python). Mais le concept est le même.

🔭 9. Portée des variables (Scope)

La portée d'une variable définit où elle est accessible dans le code. Une variable déclarée dans un bloc {} n'existe que dans ce bloc.

Locale
// x n'existe que dans main int main(void) { int x = 5; printf("%i", x); // OK } // x inaccessible ici
Globale
// Accessible partout int compteur = 0; void incrementer() { compteur++; // OK } int main(void) { compteur++; // OK }
🔀 10. Switch / case

Alternative aux if/else if quand on compare une variable à plusieurs valeurs fixes. Plus lisible dans ce cas.

int jour = 3; switch (jour) { case 1: printf("Lundi\n"); break; case 2: printf("Mardi\n"); break; case 3: printf("Mercredi\n"); break; default: printf("Autre jour\n"); }
⚠️Le break est obligatoire à la fin de chaque case. Sans lui, l'exécution "tombe" dans le case suivant (fall-through).

⚡ 6. Index — accélérer les recherches

Sans index, une recherche sur une colonne = O(n) (scan complet). Un index crée une structure interne (souvent un B-tree) qui accélère la recherche à O(log n).

-- Créer un index sur la colonne "nom" CREATE INDEX idx_nom ON etudiants (nom); -- Supprimer un index DROP INDEX idx_nom; -- Un PRIMARY KEY crée automatiquement un index
💡 Les index accélèrent les SELECT mais ralentissent les INSERT/UPDATE/DELETE (il faut mettre à jour l'index). À utiliser sur les colonnes souvent recherchées.
🛡️ 7. Sécurité — SQL Injection

Une injection SQL = un utilisateur malveillant insère du code SQL dans un champ pour manipuler la base. Très dangereux.

❌ Dangereux
-- Concatenation directe query = "SELECT * FROM users WHERE nom = '" + nom + "'" -- Si nom = "' OR '1'='1" -- → retourne TOUT !
✅ Sécurisé
-- Paramètre lié (placeholder ?) db.execute( "SELECT * FROM users WHERE nom = ?", (nom,) )
⚠️ En CS50 Python, toujours utiliser des placeholders ? avec db.execute(). Ne jamais concaténer directement des valeurs dans une requête SQL.

🧪 1. C'est quoi Flask ?
🔗 Flask combine tout ce qu'on a appris
W6 Python : la syntaxe de base → W7 SQL : les requêtes de données → W8 HTML/CSS/JS : le frontend →

Flask est un micro-framework web Python. Il permet de créer des sites web dynamiques en combinant Python (logique), HTML/CSS (affichage) et SQL (données).

📌 Flask = backend. Le code Python tourne sur le serveur, pas dans le navigateur (contrairement à JavaScript).

🐍 3. Le fichier app.py — Routes

Une route associe une URL à une fonction Python. Quand le navigateur visite l'URL, Flask exécute la fonction et renvoie la réponse.

from flask import Flask, render_template, request, redirect, session from cs50 import SQL app = Flask(__name__) app.secret_key = "ma_clé_secrète" # pour les sessions db = SQL("sqlite:///ma_base.db") # Route simple — GET @app.route("/") def index(): etudiants = db.execute("SELECT * FROM etudiants") return render_template("index.html", etudiants=etudiants) # Route avec GET et POST @app.route("/login", methods=["GET", "POST"]) def login(): if request.method == "POST": nom = request.form.get("nom") if not nom: return render_template("login.html", error="Nom requis") session["user"] = nom return redirect("/") return render_template("login.html") # Route avec paramètre dynamique @app.route("/etudiant/<int:id>") def etudiant(id): row = db.execute("SELECT * FROM etudiants WHERE id = ?", id) return render_template("etudiant.html", etudiant=row[0])
Fonction FlaskRôle
render_template()Rend un fichier HTML du dossier templates/
request.form.get()Récupère un champ d'un formulaire POST
request.args.get()Récupère un paramètre d'URL (?q=...)
redirect()Redirige vers une autre URL
session[]Stocke des données entre les requêtes
url_for()Génère l'URL d'une route par son nom
jsonify()Renvoie une réponse JSON (pour les API)
📨 4. GET vs POST
GET

Les données sont dans l'URL : ?nom=Alice&age=20. Visible, bookmarkable. Pour lire des données.

POST

Les données sont dans le corps de la requête. Invisible dans l'URL. Pour envoyer des données (login, formulaire…).

# GET : paramètre dans l'URL (/recherche?q=python) @app.route("/recherche") def recherche(): q = request.args.get("q", "") results = db.execute("SELECT * FROM cours WHERE titre LIKE ?", f"%{q}%") return render_template("results.html", results=results, query=q) # POST : données du formulaire HTML @app.route("/inscription", methods=["GET", "POST"]) def inscription(): if request.method == "POST": nom = request.form.get("nom") email = request.form.get("email") db.execute("INSERT INTO users (nom, email) VALUES (?, ?)", nom, email) return redirect("/") return render_template("inscription.html")
⚠️ Toujours utiliser POST pour les mots de passe et données sensibles. Ne jamais mettre un mot de passe dans l'URL (GET).

🔐 6. Sessions — garder l'utilisateur connecté

HTTP est sans état : chaque requête est indépendante. Les sessions permettent de retenir des infos d'une page à l'autre (ex : utilisateur connecté).

from flask import session, redirect # Stocker dans la session (lors du login) session["user_id"] = 42 session["username"] = "Alice" # Lire depuis la session user_id = session.get("user_id") # Vérifier si connecté if "user_id" not in session: return redirect("/login") # Supprimer la session (déconnexion) session.clear()
💡 La session est stockée dans un cookie chiffré dans le navigateur. Le secret_key de Flask sert à chiffrer ce cookie — il doit être long et aléatoire en production.
🔌 7. API JSON avec Flask

Flask peut aussi servir des API JSON — des routes qui renvoient des données plutôt que du HTML. Utile pour communiquer avec JavaScript côté navigateur.

from flask import jsonify @app.route("/api/etudiants") def api_etudiants(): etudiants = db.execute("SELECT * FROM etudiants") return jsonify(etudiants) # → renvoie du JSON : [{"id": 1, "nom": "Alice", ...}, ...]
// JavaScript : appeler l'API Flask fetch("/api/etudiants") .then(r => r.json()) .then(data => { data.forEach(e => console.log(e.nom)); });
